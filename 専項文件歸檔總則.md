## 檔案分類規範

### 優先級 1️⃣：**核心開發檔案** 📂 `root` 或具體功能文件夾
**特徵**：項目運行必需，頻繁修改  
**存放位置**：項目根目錄 或 `/backend` `/frontend` `/supabase`

| 檔案類型 | 命名規則 | 例子 | 存放位置 |
|---------|---------|------|---------|
| 配置文件 | `config.*` `*.config.*` | `tsconfig.json` `supabase/config.toml` | 功能文件夾根目錄 |
| 環境變數 | `.env*` | `.env.local` `.env.example` | 功能文件夾根目錄 |
| 啟動/構建腳本 | `setup-*.sh` `build.sh` | `setup-phase-one.sh` | 項目根目錄 |
| 源代碼 | `*.ts` `*.tsx` `*.js` 等 | `App.tsx` `supabase.ts` | 對應功能文件夾 |
| 依賴文件 | `package.json` `package-lock.json` | `package.json` | 功能文件夾根目錄 |

---

### 優先級 2️⃣：**文件夾結構檔案** 📂 `/docs` 子文件夾
**特徵**：按功能/主題細分的文檔，較少修改  
**存放位置**：`/docs` 中的子文件夾

| 子類別 | 文件夾名 | 文件類型 | 說明 |
|-------|---------|--------|------|
| 系統架構 | `/docs/專案架構說明` | `.md` | 技術方案、選型建議、架構圖 |
| 產品文檔 | `/docs/產品概述及使用場景說明` | `.md` | 產品需求、用戶場景、SOP手冊 |
| 開發日誌 | `/docs/每日開發進度追蹤與報告` | `.md` | 日報、迭代記錄、變更日誌 |
| **新增：部署指南** | **`/docs/部署與上線指南`** | `.md` | 三階段部署說明、檢查清單 |
| **新增：API文檔** | **`/docs/API文檔`** | `.md` | 接口說明、Swagger文檔 |

---

### 優先級 3️⃣：**快速參考檔案** 📂 `root`
**特徵**：入門指南、快速啟動、全局檢查清單  
**存放位置**：項目根目錄

| 用途 | 檔案名 | 對象 | 何時修改 |
|------|-------|------|---------|
| 📖 **入門首頁** | `START_HERE.md` | 新開發者 | 阶段更新時 |
| 🚀 **快速啟動** | `QUICK_START.md` | 急於開始 | 工具/版本更新 |
| 📋 **項目概況** | `README.md` | 所有人 | 定期（月） |
| ✅ **階段檢查清單** | `PHASE_ONE_CHECKLIST.md` 等 | 項目經理 | 階段完成時 |
| 📊 **項目總結** | `PHASE_ONE_SUMMARY.md` | 技術主管 | 階段完成時 |
| 🎯 **最終報告** | `FINAL_REPORT.md` | 項目幹系人 | 階段完成時 |

---

### 優先級 4️⃣：**配置和工具檔案** 🔧 `root` 或隱藏文件夾
**特徵**：系統配置、工具設置，很少修改  
**存放位置**：項目根目錄（`.` 開頭隱藏）或對應文件夾

| 檔案夾 | 內容 | 說明 |
|-------|------|------|
| `.github/` | workflows 等 | GitHub 工作流配置 |
| `.claude/` | MCP 配置 | Claude/編輯器專用配置 |
| `.obsidian/` | Obsidian 設置 | 筆記軟件配置 |
| `建物謄本PDF範例/` | 樣本文件 | 參考數據（可移至 `/docs/樣本與參考`) |

---

### 優先級 5️⃣：**外部資源和資料** 📦 `/resources` 或 `/data`
**特徵**：數據文件、圖片、Excel表等  
**建議新建文件夾**

| 資源類型 | 文件夾 | 說明 |
|---------|-------|------|
| 樣本文件 | `/resources/samples` | PDF、圖片範例 |
| 數據表 | `/resources/data` | Excel 地號對照表等 |
| 靜態資源 | `/frontend/assets` | Logo、圖標（已有） |
| 認證密鑰 | **不入版本庫** | `.gitignore` 中排除 |

## 文件命名規則

### 🏷️ Markdown 檔案命名

| 類型 | 規則 | 例子 |
|------|------|------|
| **一般文檔** | `描述性名稱.md` | `專案架構說明.md` `使用者場景.md` |
| **日期相關** | `YYYY-MM-DD_描述.md` | `2026-01-13_進度報告.md` |
| **檢查清單** | `[名稱]_檢查清單.md` 或 `CHECKLIST_[名稱].md` | `部署檢查清單.md` |
| **API 文檔** | `[服務名]_API.md` | `認證API.md` |
| **根目錄快速參考** | 全大寫或 `START_HERE` 格式 | `README.md` `QUICK_START.md` |

### 🔤 其他檔案命名

| 類型 | 規則 | 例子 |
|------|------|------|
| **啟動腳本** | `setup-[階段].sh` | `setup-phase-one.sh` |
| **配置文件** | `[功能名].config.js` 或 `.env.local` | `tsconfig.json` |
| **資料文件** | 中文描述性名稱 | `大安區地號建物名稱對照表.xlsx` |

## 優先級和存放位置

### 快速決策表：我的文件應該放在哪裡？

```
1. 是代碼文件？
   ├─ 是 → 放在 /backend /frontend /supabase 中
   └─ 否 → 進入下一步

2. 是系統配置或工具配置？
   ├─ 是 → 放在根目錄或隱藏文件夾（.github .claude 等）
   └─ 否 → 進入下一步

3. 是部署或上線相關？
   ├─ 是 → 放在 /docs/部署與上線指南
   └─ 否 → 進入下一步

4. 是產品或業務相關？
   ├─ 是 → 放在 /docs/產品概述及使用場景說明
   └─ 否 → 進入下一步

5. 是架構或技術方案？
   ├─ 是 → 放在 /docs/專案架構說明
   └─ 否 → 進入下一步

6. 是開發日誌或進度？
   ├─ 是 → 放在 /docs/每日開發進度追蹤與報告
   └─ 否 → 進入下一步

7. 是新手入門相關？
   ├─ 是 → 放在根目錄（START_HERE.md QUICK_START.md）
   └─ 否 → 進入下一步

8. 是樣本、數據、圖片？
   ├─ 是 → 放在 /resources/samples 或 /resources/data
   └─ 否 → 詢問技術主管

```

## 文件生命週期管理

### 📅 文件維護計劃

| 檔案類型 | 更新頻率 | 責任人 | 備註 |
|---------|---------|-------|------|
| 代碼文件 | 實時 | 開發者 | 代碼審查 |
| README.md | 月度 | 技術主管 | 與版本同步 |
| QUICK_START.md | 工具/版本更新時 | 開發者 | 驗證步驟有效性 |
| 架構文檔 | 季度/需求變更 | 技術主管 | 評估後更新 |
| API 文檔 | API 變更時 | 開發者 | 同步發佈 |
| 日進度報告 | 每日（可選） | 開發者 | 存檔備查 |
| 階段檢查清單 | 階段完成時 | 項目經理 | 歸檔保存 |
| 設定檔案 | 需要時 | 相關開發者 | 同步更新 .env.example |

### 🗑️ 文件歸檔和清理

```
不活躍文件（6+ 個月未修改）
├─ 檢查是否過時
├─ 如已過時 → 移至 /docs/archived/ 並標記 [DEPRECATED]
└─ 如仍有參考價值 → 保留但在文件頭添加 [ARCHIVED - Last Updated: YYYY-MM-DD]

垃圾文件或測試文件
├─ 檢查 .gitignore 規則
└─ 確保不提交至版本控制
```

## 📞 常見問題

**Q: 既然文檔都在 `/docs`，為何還需要根目錄的 `START_HERE.md` 等？**  
A: 根目錄文件是"快速入口"，新人打開項目會看到。它們應該簡潔，指向詳細文檔。

**Q: 圖片和截圖應該放在哪裡？**  
A: 
- 前端 UI 相關 → `/frontend/assets`
- 文檔插圖 → `/docs/[相應子文件夾]/assets` 或同文件夾中
- 樣本截圖 → `/resources/samples`

**Q: 如何處理過期文檔？**  
A: 不要直接刪除。在文件頭添加 `[已過期] 最後更新: YYYY-MM-DD`，或移至 `/docs/archived` 備查。

**Q: 日進度報告應該多久整理一次？**  
A: 建議每周末將日報歸檔。月底生成月度總結。
